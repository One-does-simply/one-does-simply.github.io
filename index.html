One Does Simply

A family of spec-driven frameworks

A builder writes a JSON spec describing their app; a framework renders it as a fully functional application with persistence, auth, and a polished UI.

Families

Each family is a self-contained monorepo — the specification, its renderers, the AI build helpers that author it, and conformance scenarios that pin behavior across implementations.

Philosophy

  • Specs over code. If behavior can be described declaratively in JSON, it should be.
  • Multiple renderers per family. One spec, many targets — web, mobile, desktop.
  • Atomic cross-framework commits. When the spec changes, all renderers update together.
  • No silent breaking changes. Conformance scenarios pin behavior; renderers that disagree fail loudly.
